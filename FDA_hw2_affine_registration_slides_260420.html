Affine Linear Registration의 구현 이슈

Numerical Stability & Constraints

통계학과 신현수
1. 문제 설정
  • Shift Registration (기존): \(h(t) = t + \delta\)
  • Affine Warping (확장): \(h(t) = a + bt\)
\[ F_{\text{linear}}(a,b) = \int_{T_1}^{T_2} \{x(a+bt) - x_0(t)\}^2 \, dt \]
  • 추정 방법: Newton-Raphson 알고리즘으로 \((a, b)\)를 반복 추정
2. 구현 이슈 1 — 시간 순서 보존
  • Warping 함수 \(h(t) = a + bt\)의 기울기: \(h'(t) = b\)
  • 수학적 제약: 시간은 역행할 수 없으므로 반드시 \(b > 0\)을 만족해야 함
Newton-Raphson의 full step 적용 시 \(b \le 0\)이 되어 시간 순서가 역전되는 문제 발생 가능
3. 해결책 — Step-halving
Backtracking line search의 가장 단순한 형태로, 탐색 방향은 고정한 채 step size \(\alpha\)를 반복적으로 절반씩 줄여 제약 조건과 수렴을 동시에 보장하는 방법
  • Newton 방향은 유지하되, step size \(\alpha\)를 조절하여 제약 조건을 보장
\[\begin{pmatrix} a_{\text{new}} \\ b_{\text{new}} \end{pmatrix} = \begin{pmatrix} a \\ b \end{pmatrix} - \alpha H^{-1}g\]
  • \(\alpha = 1\)에서 시작
  • \(b_{\text{new}} \le 0\)이거나 목적 함수 값이 증가하면 \(\alpha \leftarrow \alpha / 2\)
  • 조건을 만족할 때까지 반복
4. 코드 구현 — Step-halving

Step-halving 과정

1. Newton이 제안한 방향과 크기 그대로 이동해본다
2. 결과가 나쁘면 보폭을 절반으로 줄이고 다시 시도
3. 괜찮아질 때까지 반복   \(\alpha = 1 \to 0.5 \to 0.25 \to \cdots\)

R 구현 예시

# b > 0 제약 조건 및 Step-halving 구현
anew <- a - stepsize * step[1]
bnew <- b - stepsize * step[2]

if (bnew <= 0) {
  stepsize <- stepsize / 2
  if (stepsize < 1e-6) break
  next
}
5. 구현 이슈 2 — Hessian의 수치적 불안정성
  • 발생 조건: 곡선이 완만하거나 뚜렷한 특징(peak/valley)이 없는 경우
  • 결과: Hessian 행렬 \(H\)가 nearly singular에 가까워짐
\(H^{-1}\) 계산 시 수치 오차가 증폭되어 추정값이 발산하거나 수렴에 실패
6. 해결책 — Ridge Term
  • Hessian의 대각 원소에 작은 상수 \(\epsilon\)을 더해 행렬을 양정치(positive definite)로 안정화
\[ H \leftarrow H + \epsilon I \]

R 구현 예시

H <- FLHess(t0, x0, t, x, a, b, ngrid = ngrid)
H <- H + diag(1e-8, 2)  # 작은 ridge term 추가
step <- drop(solve(H, g))
7. 결론
Step-halving

\(b > 0\) 제약을 보장하여 수학적 타당성 확보 — Newton 방향을 유지하면서 단계적으로 step size를 줄임

Ridge Term

Hessian이 nearly singular일 때 \(\epsilon I\)를 더해 역행렬 계산을 안정화하고 발산을 억제

최적화 알고리즘의 수렴만큼이나, 제약 조건과 수치적 안정성을 함께 다루는 것이 중요함
